Anti Dcode

I’ve heard that everyone just uses dcode.fr to solve all of their crypto problems. Shameful, really.

This is really just a basic Caesar cipher, with a few extra random characters on either side of the flag. Dcode can handle that, right? >:)

This challenge was just about doing a caesar decrypt on the right amount of shift on the flag that was somewhere in the data given. So as the data given was big i tried to find an occurence of a shifted “utflag{” because the flag was going to start by this string:

text = "data_from_the_challenge"

alphabet = ['a', 'b', 'c', 'd', 'e', 'f', 'g', 'h', 'i', 'j', 'k', 'l', 'm', 'n', 'o','p', 'q', 'r', 's', 't', 'u', 'v', 'w', 'x', 'y', 'z']

def caesar_list(s):
    res = []
    for shift in range(26):
        temp = ""
        for char in s:
            if char >='a' and char<='z':
                char = alphabet[(alphabet.index(char) + shift) % 26]
            temp += char
        res.append(temp)
    return res

variations = caesar_list("utflag{")

for i in range(len(variations)):
    if variations[i] in text:
        print(i)
        break

And this found an 8 times shifted “utflag{“which was “cbntio{”. Then I just found the string corresponding to the flag in the data :

cbntio{zqx_lkwlm}

and applied a shift of -8 to recover the flag :

utflag{rip_dcode}
